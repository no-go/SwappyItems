44 /*
45  * ## Details
46  *
47  * - 2 phase pairing heap
48  * - the heap magic is in `update()`, `del()` and the private methode `insert()`
49  * - additional to a Item(key, value, vector of keys, prio) is stored (private):
50  * - a parent key
51  * - a 2nd vector of keys to "sibling nodes"
52  * - a head key and its data (outside of _ramList and files on DISK)
53  *
54  * ## Interesting
55  *
56  * - most recently used elements are still in RAM
57  * - elements with less access in PRIO updates may Swap to DISK (= deep heap content leaves the RAM!)
58  *
59  * @tparam TKEY is the type for the key, which identifies the Item.
60  * @tparam TVALUE is the type for the value, which has the data of the Item.
61  *
62  * @tparam EACHFILE is number of Items in each swap file. Please choose a 2^n number like 65536 or 2048
63  * @tparam OLDIES is number of files, if a swap occurs. Please choose a 2n number above 6
64  * @tparam RAMSIZE if _ramList has more than RAMSIZE x OLDIES x EACHFILE Items, a swap occurs.
65  * @tparam BLOOMBITS is the number of fingerprint bits for a key, which are set in a MASK for each file. Stupid value: 1 Bit at MASKLENGTH = EACHFILE; possible but not very meaningful: 4 Bits at MASKLENGTH = 4x EACHFILE
66  * @tparam MASKLENGTH is the MASK size of a file, where the fingerprint bits are set. A good choice to get 50% filling: 2x BLOOMBITS x EACHFILE
67  */

70  class TVALUE,
71 
72  int EACHFILE = 16384,
73  int OLDIES = 8,
74  int RAMSIZE = 3,
75  int BLOOMBITS = 4,
